Development

Detailed overview of how to get the system compiled.

(?)Ubuntu Linux

An up-to-date (read: latest) version of some form of ubuntu is strongly recommended. Because we do alot of with with the Qt Toolkit, we recommend Kubuntu, but any up to date linux distro should work.

Packages

List of packages to install before being able to compile anything

Common

These packages are used across many/all of the components

  • cmake
  • qt4-dev-tools
  • g++
  • libcv-dev
  • libboost-dev
  • cantlr
  • libantlr-dev
  • libblas-dev
  • liblapack-dev

sudo apt-get install cmake subversion qt4-dev-tools g++ libcv-dev libboost-dev cantlr libantlr-dev

Vision

The vision program requires some extra packages

  • libavformat-dev
  • libswscale-dev
  • freeglut-dev
  • libdc1394-22-dev

sudo apt-get install libavformat-dev libswscale-dev freeglut-dev libdc1394-22-dev

Radio

  • libusb-dev

sudo apt-get install libusb-dev

Simulator

Simulator uses the NVidia PhysX physics library and must be cross compiled in 32-bit mode on 64bit systems.

32 bit

  1. Change to the lib dir in the simulator directory
  2. Install the packages (.deb) files

sudo dpkg -i *.deb

64 bit

  1. Change to the lib dir in the simulator directory
  2. Install the packages (.deb) files (using --force-architecture)
  3. Install additional packages for cross compiling
  4. Check that libQtOpenGL.so exists in /usr/lib32
    - if it does not, then get it from a 32bit deb package for your distro and put it there. (Make sure to get all the libQtOpenGL.so files not just the symlinks)
  5. Make sure that other Qt libraries in /usr/lib32 have proper symlinks.
    - (i.e. libQtCore.so links to libQtCore.so.# which links to libQtCore.so.#.# .. etc
    - If this is not the case run make linkfix from the cmake build directory

sudo dpkg --force-architecture -i *.deb

sudo apt-get install libc6-dev-i386 g++-multilib ia32-libs

CMake

CMake is a cross platform build system. We use it primary because of its dependency solving capabilities and to simplify building Qt applications. CMake build files are called CMakeLists.txt and are often found in the root directory of a particular application or program within our source tree. In each application directory there is a build folder (usually bin or build. It is used for out-of-source builds and all of the executables and intermediate files from CMake go in there.

Quick Start

  1. Navigate to the build folder for the particular application you wish to build
  2. type: cmake .. on the command line.
    - This generates the makefile(s).
  3. type: make
    - This builds the code.

Do this one: cmake .. && make
